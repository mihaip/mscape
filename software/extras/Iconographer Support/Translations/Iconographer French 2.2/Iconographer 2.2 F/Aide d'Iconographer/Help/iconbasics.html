Tailles d'une icone

    Les icones peuvent être visualisées à différentes tailles, comme dans le Finder. Vous pouvez décider de ne créer qu'une taille et de laisser le logiciel créer les autres automatiquement, mais le résultat est meilleur si vous créez vous-même chaque taille, afin de donner à chacune la meilleure qualité visuelle. Cependant, si vous décidez de ne pas inclure certaines tailles, il vaut mieux créer les grandes et ignorer les petites, car une réduction d'icone donne un meilleur résultat qu'un aggrandissement. Voici toutes les tailles supportées par Iconographer :


    Mini

    Dimensions:16 x 12
    Profondeurs :8-bit, 4-bit, 1-bit, masque 1-bit
    Description: La seule icone qui n'est pas carrée. Rarement utilisée, c'est en particulier le format du plus petit affichage en liste sous MacOS 8 et 9.
    Formats supportés : Mac OS Ancien, Mac OS Nouveau, Mac OS Universel.


    Petite

    Dimensions:16 x 16
    Profondeurs :32-bit, 8-bit, 4-bit, 1-bit, 8-bit mask, masque 1-bit
    Description: La taille standard de l'affichage en liste. Egalement utilisée dans la barre des menus et comme icone de proxy.
    Formats supportés : Mac OS Ancien, Mac OS Nouveau, Mac OS Universel, Windows, Windows XP, Mac OS X Server.


    Grande

    Dimensions:32 x 32
    Profondeurs :32-bit, 8-bit, 4-bit, 1-bit, 8-bit mask, masque 1-bit
    Description: C'est la taille traditionnelle. La taille par défaut avant MacOS X.
    Formats supportés : Mac OS Ancien, Mac OS Nouveau, Mac OS Universel, Windows, Windows XP, Mac OS X Server.


    Très grande

    Dimensions:48 x 48
    Profondeurs :32-bit, 8-bit, 4-bit, 1-bit, 8-bit mask, masque 1-bit
    Description: Même si elle est principalement utilisée par MacOS X Server, cette taille est aussi utilisable sous MacOS, en utilisant la réduction d'icone.
    Formats supportés : Mac OS Nouveau, Windows, Windows XP, Mac OS X Server.


    Enorme

    Dimensions:128 x 128
    Profondeurs :32-bit, masque 8-bit
    Description: La taille standard sous MacOS X. Souvent réduite à une taille plus pratique.
    Formats supportés : Mac OS Nouveau

Profondeur des icones

    Pour chaque taille d'icone, il peut y avoir différentes profondeurs de couleurs. Cela signifie que vous pouvez avoir une icone en noir & blanc, une en 16 couleurs, une autre en 256 et une dernière en millions de couleurs. Le Finder choisit automatiquement la meilleure en fonction des capacités de l'écran. Là aussi, vous pouvez simplement en dessiner une et laisser le système créer les autres, mais les résultats sont meilleurs si vous décidez de dessiner chacune d'elles.

Masques d'icones

    Il n'est pas évident de voir que chaque icone est composée de deux éléments : l'image elle-même et le masque. Le masque agit comme un découpage déterminant quels pixels afficher et quels pixels cacher. L'illustration ci-dessous montre le mécanisme d'un masque d'icone :

    Comme vous pouvez le voir, le fond d'écran est visible là où le masque est blanc. Le masque de base n'a que deux valeurs : blanc (où le fond d'écran est visible) et noir (où l'icone s'affiche). MacOS 8.5 a introduit un masque supplémentaire, le masque 8-bits. Ce nouveau masque autorise à utiliser 256 valeurs, du blanc au noir. Comme avant, le blanc signifie que le fond d'écran est visible, et le noir signifie que l'icone s'affiche. Les valeurs intermédiaires de gris sont utilisées pour mélanger le fond d'écran et l'icone. Les masques 8-bits peuvent être utilisés pour créer des effets visuels, tels que transparences, anti-aliasing (lissage) ou ombres portées.

    Les masques 8-bits sont supportés par les formats d'icones Mac OS 8.5+, Mac OS Universel et Mac OS X, Mac OS X Server et Windows XP.

Formats d'icones

    Ancien (ressources) : C'est le format utilisé par les systèmes avant MacOS 8.5. Ils n'acceptent que les icones de 1 à 8 bits, en 32x32 ou 16x16 pixels. Ce format reste requis pour un maximum de compatibilité des icones (MacOS 8.5 et suivants savent encore afficher ces icones) en taille minimum.

    Nouveau (ressources) : C'est le format introduit par MacOS 8.5. Il supporte les icones en 32 bits avec un masque 8 bits, ainsi que la nouvelle taille de 48x48 pixels (qui, s'il n'est pas utilisé par le Finder, apparaît à d'autres occasions, comme dans Sherlock 2). Ces icones utilisent un mode d'enregistrement tout à fait différent et ne peuvent pas être visualisées avec les vieux systèmes.

    Universel (ressources) : C'est un pseudo-mode qui combine les anciennes et nouvelles ressources dans le même fichier, avec le même identifiant. Ainsi, l'icone est visible sur tous les systèmes, mais le résultat sera meilleur quand l'icone sera visualisée sous MacOS 8.5 et suivants. Ce format accroit légèrement la taille des fichiers.

    Mac OS X (fichier .icns) : Mac OS X supporte les icones du format Nouveau, avec en plus la taille 128 x 128. De plus, en raison de l'abandon progressif du Ressource Fork, les icones sont maintenant conservées dans un fichier indépendant, avec le suffixe .icns. Bien que ce soit le format d'icones natif de MacOS X, il faut noter que le Finder de MacOS X ne sait en fait pas les afficher, elles s'afficheront avec une icone générique de fichier. Si vous voulez les voir, vous devez les ouvrir dans Iconographer, Icon Composer, ou un autre éditeur d'icones.
